## Documentation Types

### Living Documentation (architecture/)
Documents that describe the current state of the system architecture. These should be updated when the architecture changes.

### Reports (reports/)
Point-in-time analysis reports, audit summaries, and testing reports. These are historical snapshots and are not updated.

### Archive (archive/)
Historical development artifacts, session summaries, and obsolete planning documents. These are kept for reference but are not actively maintained.

### Guides (guides/)
Developer guides, onboarding documentation, and how-to articles (to be added).
